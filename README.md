# E-commerce Web QA Jira Project

## Project Goal

This project demonstrates manual QA testing of an existing e-commerce demo application.

The goal is to simulate a real QA workflow:

- test strategy preparation;
- checklist creation;
- test case design;
- manual test execution;
- bug reporting in Jira;
- evidence collection;
- test summary preparation;
- basic regression planning.

## Application Under Test

Application: nopCommerce Demo Store  
URL: https://demo.nopcommerce.com/  
Application type: E-commerce demo web application

Main tested areas:

- home page;
- product catalog;
- product details page;
- search;
- cart;
- checkout flow;
- registration and login;
- form validation;
- mobile responsiveness;
- browser console and network behavior.

## QA Scope

The project covers:

- functional testing;
- UI testing;
- validation testing;
- negative testing;
- exploratory testing;
- basic regression testing;
- Jira-style bug tracking;
- evidence-based bug reporting.

## Out of Scope

The following areas are not covered:

- performance testing;
- security testing;
- load testing;
- payment gateway testing;
- penetration testing;
- automated testing;
- real production data testing.
